   `flutter run -d chrome`

## Ответы на вопросы

### 1. Зачем выносить виджеты в отдельные файлы? Что изменится, если держать всё в main.dart?

Вынесение виджетов в отдельные файлы повышает читаемость, поддерживаемость и переиспользуемость кода. Каждый виджет находится в своём файле, что упрощает навигацию по проекту, особенно когда приложение растёт. Если держать всё в `main.dart`, файл быстро превращается в «свалку» из сотен строк, его сложно читать и отлаживать. Кроме того, изолированные виджеты легче тестировать и повторно использовать в других частях приложения или даже в других проектах.

### 2. Что такое BuildContext? Почему метод build() принимает его как параметр?

`BuildContext` — это объект, который содержит информацию о местоположении виджета в дереве виджетов. Через него можно получить доступ к родительским виджетам, теме, медиа-запросам (размер экрана) и многому другому. Метод `build()` принимает `BuildContext`, потому что при построении UI виджету нужно знать своё окружение: например, чтобы правильно применить наследуемые стили или найти вышестоящий `Scaffold`. Flutter передаёт этот контекст автоматически при каждой перерисовке.

### 3. Чем StatelessWidget отличается от StatefulWidget? Приведите пример, когда нужен каждый из них.

| Характеристика | StatelessWidget | StatefulWidget |
|----------------|----------------|----------------|
| Состояние       | Не имеет внутреннего изменяемого состояния | Имеет состояние, которое хранится в объекте `State` |
| Перерисовка     | Только при изменении внешних параметров | По вызову `setState()` внутри состояния |
| Пример          | Статическая иконка, текст, контейнер с фиксированным градиентом | Счётчик кликов, форма ввода, игральный кубик, анимация |

**Пример:**  
- `StatelessWidget` – виджет `StyledText`, который просто отображает переданный текст. Он не меняется после создания.  
- `StatefulWidget` – виджет `DiceRoller`, который при нажатии кнопки меняет изображение кубика. Ему нужно помнить текущую грань и обновлять UI.

### 4. Почему Random() создаётся на уровне файла, а не внутри rollDice()?

Создание `Random()` на уровне файла позволяет использовать один и тот же генератор случайных чисел на протяжении всего времени работы приложения. Это более эффективно, так как не требует создания нового объекта при каждом нажатии кнопки.
